Less modern dev

  • Site built using a text editor and web browser
  • Some amount of jQuery
  • No use of a library such as Backbone or a framework such as AngularJS
  • Poor testing practices
  • Raw files deployed to webserver without packaging
--- > In order to add context let's consider what is not considered modern web development > Then we'll look at tools and processes which build on a basic foundation to create a more modern approach > So, not too long ago sites were built using tools like Dreamweaver or basic text editors. The sites were essentially HTML, CSS and JavaScript with sprinklings of jQuery. > Less modern sites don't use of a library or framework which would help to structure code for maintainability. > Less modern sites don't include any unit testing of JavaScript code

More modern dev

  • Seperation of concerns
  • Model, View, Controllers
  • Libraries and frameworks to add structure and improve maintainability
  • Tools, called linters, that help spot issues and support maintaining best practices
--- > A good way to understand modern web development is to understand what problems it tries to solved > So, not too long ago sites were built using tools like Dreamweaver or basic text editors. The sites were essentially HTML, CSS and JavaScript with sprinklings of jQuery. > Web projects have become increasingly complex. A large web application might include dozens of programmers and an almost equal part of designers, user interface and experience experts, QA, product and project managers. > Some web applications take an army to create and maintain.

JavaScript, front to back and sideways

  • No longer only in the front-end
  • JavaScript run in the back-end on NodeJS
  • Also used to build database queries in CouchDB and CouchBase
  • JavaScript Object Notation is also used with databases such as Couch, Mongo and Redis
  • JS is also being used in Robotics, checkout Cylon.js and NodeBots
--- > JavaScript on the front end > JS in the backend with NodeJS > JS in a database with CouchDB, CouchBase > JS with MongoDB and Redis > JS in robotics

Learn JavaScript

No really, Learn JavaScript!
